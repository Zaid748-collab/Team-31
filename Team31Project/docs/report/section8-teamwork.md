## Teamwork Ethics

### Communication

Our team used Discord for daily communication and task coordination. As shown in Appendix A, we maintained active discussions about feature implementation, UI improvements, and report progress. Key decisions were made collaboratively, with team members providing feedback on each other's work.

For example, when discussing UI improvements, @BizyBeeV suggested enhancing the call-to-action (CTA) and navigation labels (see Discord 03/02/2026). This open feedback culture helped us refine our design iteratively.

### Task Management and Progress Tracking

We adopted an agile-inspired workflow using Trello to manage our project backlog (see Appendix B). Tasks were organized into Term 1 and Term 2 columns, with clear ownership and status indicators. My dark mode implementation was tracked from "To Do" to "Done," demonstrating our commitment to visible progress tracking.

Progress was also monitored through team check-ins. As shown in Discord (10/03/2026), Harvey tracked each member's status on coding vs report completion:
- **Joshua**: Coding ✓, Report ✓
- **Harvey**: Coding ✗, Report ✓
- **Zaid**: Coding ✓, Report ✓
- **Leon**: Coding ✓, Report ✓
- **Kane**: Coding ✓, Report ✓
- **Steve**: Coding ✓, Report ✗
- **Mohamed**: Coding ✓, Report ✓

This transparency helped us identify gaps and ensure balanced contributions.

### Code Management and Version Control

All code was managed via GitHub using a feature-branch workflow. My dark mode work was developed on the 'light/dark-mode' branch and merged after team review. Git commit history (Appendix C) shows regular contributions with clear messages like "Implemented dark/light mode with smooth purple gradient," demonstrating disciplined version control practices.

### Quality Assurance and Testing

During testing, I identified contrast issues on product detail pages where text was not visible in dark mode. After investigation, I implemented CSS fixes and documented the before/after results (see Figure 4b in testing section and Appendix D). This iterative improvement process demonstrates our commitment to accessibility and user experience.

### Collaborative Decision-Making

Design decisions were guided by established principles. As @minion noted in Discord (03/02/2026): *"every decision we make should be based on some sort of principles etc, that goes for ui testing etc will make the report a whole lot easier to discuss."* This approach ensured our UI choices were justified and aligned with best practices, such as the 60-30-10 colour rule for visual balance.

### Conflict Resolution

When CSS conflicts arose during dark mode implementation, we conducted paired programming sessions to resolve them collaboratively. Regular weekly meetings ensured that blockers were addressed promptly and that all team members remained aligned on project goals.
